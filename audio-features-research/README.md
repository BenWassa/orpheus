# Audio Features Research Package

**Problem:** Orpheus needs valence, arousal, and depth (V/A/D) values for 4,243 Spotify
tracks to power its GMM clustering step. Currently `audio_features` has 0 rows, so the
clustering section of every report outputs `clusters_status: "no_audio_features"`.

This folder contains everything needed to research and evaluate solutions externally.

## Quick orientation

The pipeline step that needs audio features: `orpheus/enrich/enrich.py` → `ArchiveReader`
lookup → writes into `audio_features` table.

The downstream consumer: `orpheus/pattern/cluster.py` → `_load_avd_data()` joins
`track_scores` and `audio_features` on `track_uri`.

Required fields (minimum viable): `valence`, `arousal`. Optional but used: `tempo`,
`energy`, `danceability`, `acousticness`, `instrumentalness`, `loudness`.

Resolution keys available per track: `track_uri` (Spotify URI), `ISRC` (where populated),
track name + artist name.

## Decision criteria (ranked)

1. **Coverage** — what % of 4,243 tracks does the source resolve?
2. **Cost** — free tier sufficient for a one-time 4K-track import?
3. **Effort** — bulk download vs. per-track API vs. local extraction?
4. **Maintenance** — is the source still actively maintained?

The ideal outcome is a one-time bulk import that populates the table permanently (no
ongoing API dependency). See `archive_lookup_spec.md` for the preferred path.
