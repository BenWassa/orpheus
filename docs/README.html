Understanding the static export

This GitHub Pages build is a read-only snapshot of Project Orpheus results.

Why this site exists

The exporter script in 01_setup/export_static_site.py transforms your Spotify Exportify CSV files into accessible HTML pages that you can host on GitHub Pages. It reuses the same data-cleaning and emotion analysis pipeline as the interactive Streamlit app, but the charts and summaries are pre-generated.

What you can and cannot do here

  • ✅ Browse existing datasets: Every CSV exported through the pipeline gets a dedicated analysis page with charts, metrics, and a downloadable .summary.json.
  • ✅ Share read-only insights: Friends and collaborators can explore your listening trends without installing Python.
  • ❌ Upload new files: This site does not include file upload, authentication, or live Spotify integrations. All data is generated ahead of time.
  • ❌ Refresh analytics automatically: Regenerate the static files whenever you have new playlists to analyse.

Regenerating the site

  1. Activate the Python environment that ships with the project.
  2. Place new Exportify CSV files inside 04_data/raw/.
  3. Run python 01_setup/export_static_site.py --input-dir 04_data/raw --out-dir docs --force to rebuild the HTML pages.
  4. Commit the updated docs/ folder and push to the GitHub branch powering Pages.

Need the detailed setup steps? Open 06_docs/QUICK_START.md in your local clone or on GitHub for the full walkthrough.

Tips for a better experience

  • Keep dataset names concise—the index page search matches on the title.
  • Add custom thumbnails in docs/assets/images/thumbnails/ using the dataset slug (for example, ben-rap-x.svg).
  • For a richer write-up, embed the summary JSON files into blog posts or notebooks.
  • Want interactivity? Launch the full Streamlit dashboard following the quick-start guide.
Questions? Check 06_docs/QUICK_START.md and the rest of the 06_docs/ folder locally for troubleshooting steps, or open an issue in the project repository.
